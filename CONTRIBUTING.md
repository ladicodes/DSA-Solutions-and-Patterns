# Contributing Guide

Thank you for wanting to contribute to this repository.

This project focuses on daily DSA problem-solving with clear explanations of patterns and solutions.

## Ways to contribute

- Add a new DSA solution
- Improve explanations in an existing solution
- Add alternative optimized approaches
- Fix typos, structure, or complexity notes
- Propose better folder organization

## Contribution rules

- Keep solutions educational and easy to follow
- Include the pattern used (for example: Sliding Window, Two Pointers, DFS, DP)
- Explain approach step by step
- Include time and space complexity
- Prefer clean, readable code over clever but unclear code

## Suggested file naming

Use consistent and searchable names:

- `YYYY-MM-DD-problem-name.md`
- `problem-name.<language-extension>` for code files when needed

## Recommended structure for each solution note

```md
# Problem: <name>

## Link
<problem-url>

## Pattern
<pattern-name>

## Intuition
<short explanation>

## Approach (Step by Step)
1. ...
2. ...
3. ...

## Complexity
- Time: O(...)
- Space: O(...)
```

## Pull request process

1. Fork the repository
2. Create a branch: `feat/<short-description>` or `fix/<short-description>`
3. Commit your changes with clear messages
4. Open a pull request using the PR template
5. Address feedback if requested

## Before opening a PR

- Confirm the solution runs
- Confirm explanation and complexity are included
- Confirm naming/style follows this guide
- Keep each PR focused on one problem or one small topic

Thank you for helping make this a better DSA learning resource.
